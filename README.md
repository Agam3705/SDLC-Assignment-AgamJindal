Smart-Attend: SDLC & DevOps AssignmentRole: Junior Project ManagerProject: University Student Attendance App (Geo-fencing & Biometrics)📋 Project OverviewThis repository contains the assignment deliverables for the Smart-Attend mobile application. The project focuses on automating student attendance via Geo-fencing and integrating Biometric (Face ID) verification to prevent "proxy" attendance. This documentation highlights the transition from a traditional Waterfall model to a flexible Scrum framework.📂 DeliverablesWaterfall_Analysis.pdf: A detailed analysis of why the Waterfall model failed when the new Biometric requirement was introduced mid-development.Sprint_Backlog.csv: A comprehensive list of tasks for Sprint 1 (MVP), detailing user stories, priorities, and estimated effort.🔄 The Agile Pivot (Scrum Framework)Because the Dean introduced a major new requirement (Face ID) halfway through the project, we switched from Waterfall to Scrum to handle the change effectively. The project is now organized into two-week Sprints:Sprint 1 (MVP): Focused on the "Minimum Viable Product." This included the basic UI, User Login via university ID, and the core Geo-fencing logic.Sprint 2 (The Update): Focused on integrating the Biometric (Face ID) logic and refining the reporting dashboard based on initial student feedback.⚙️ Why We Use CI/CDIn this project, CI/CD (Continuous Integration/Continuous Deployment) is critical for the following reasons:Automatic Updates: It allows us to push the new Biometric updates from Sprint 2 to students' devices automatically as soon as the code is verified.Frequent Integration: Since multiple developers work on Geo-fencing and Face ID, CI ensures their code merges without conflicts.Fast Feedback: By deploying the MVP quickly through a CD pipeline, we can catch bugs in the Geo-fencing logic before we finish the Face ID integration.
