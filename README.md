## 📋 Project Overview
[cite_start]This repository contains the assignment deliverables for the **Smart-Attend** mobile application[cite: 3]. [cite_start]The project focuses on automating student attendance via Geo-fencing and integrating Biometric (Face ID) verification[cite: 6, 7].

## 📂 Deliverables
* [cite_start]**Waterfall_Analysis.pdf**: Analysis of why traditional models fail with mid-project changes[cite: 8, 9].
* **Sprint_Backlog.csv**: Comprehensive list of tasks for **Sprint 1 (MVP)**[cite: 18, 19].

## 🔄 The Agile Pivot (Scrum Framework)
[cite_start]Because a major new requirement (Face ID) was introduced, we switched to **Scrum** to handle the change in two-week Sprints[cite: 14, 15]:

* [cite_start]**Sprint 1 (MVP):** Focused on basic UI, User Login, and core Geo-fencing logic[cite: 16].
* **Sprint 2 (The Update):** Focused on integrating Biometric (Face ID) logic and the dashboard[cite: 17].

## ⚙️ Why We Use CI/CD
In this project, **CI/CD** is essential for[cite: 27]:
1. **Automatic Updates:** Pushing Sprint updates to students automatically[cite: 27].
2. **Frequent Integration:** Ensuring new Face ID code doesn't break the Geo-fencing logic.
3. **Fast Feedback:** Catching bugs early while the team is still working on the update.
